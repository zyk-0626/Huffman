# Huffman

哈夫曼编码：文本压缩
文档：包含什么数据结构，调度方式，怎么运行

数据结构：队列、二叉树

一、需求分析（业务场景分析）
1. 项目背景
哈夫曼编码是一种无损数据压缩算法，核心目标是：

对文本数据进行压缩，减少存储空间 / 传输带宽占用
保证压缩后的数据可以完全无损还原
实现思路：给出现频率高的字符分配短编码，频率低的字符分配长编码，整体实现平均编码长度最小化

2. 功能需求
统计输入文本中各字符的出现频率
基于频率构建哈夫曼树
生成每个字符的哈夫曼编码
输出每个字符对应的编码结果

3. 业务场景示例
以代码中示例文本 abracadabra 为例：

字符频率：a:5, b:2, r:2, c:1, d:1
压缩后：原文本 ASCII 编码（每个字符 8bit）共 11×8=88bit，哈夫曼编码可压缩至更少 bit 数，实现无损压缩


二、核心数据结构设计
1. 哈夫曼树节点 
typedef struct node
{
    char ch;       // 字符（仅叶子节点有效）
    int weight;    // 权值（字符出现频率）
    struct node *left;  // 左子树（编码为0）
    struct node *right; // 右子树（编码为1）
} Node, Tree;

叶子节点：存储原始字符和对应频率，左右子树为NULL
中间节点：ch为'\0'，权值为左右子节点权值之和

2. 优先队列 
typedef struct queue
{
    Node* data[queue_size];  // 存储节点指针的数组
    int size;                // 当前队列中元素个数
} Queue;

作用：实现按权值升序排列的优先队列，保证每次取出权值最小的两个节点构建哈夫曼树
核心操作：initQueue、enqueue（按权值有序插入）、dequeue（取队首元素）


三、核心算法设计
1. 字符频率统计算法
void countFrequency(const char *text, int freq[], char chars[], int *charCount)

逻辑：遍历文本，逐个统计每个字符的出现次数，将不同字符存入chars数组，对应频率存入freq数组
时间复杂度：O(n^2)（n为文本长度，双重循环统计）

2. 哈夫曼树构建算法
Tree* buildhuffmanTree(Queue* q)

核心逻辑（哈夫曼算法）：
将所有叶子节点按权值升序加入优先队列
取出队首两个权值最小的节点，合并为一个新节点（权值为两者之和）
将新节点按权值插入优先队列
重复步骤 2-3，直到队列中只剩一个节点，即为哈夫曼树的根节点

3. 哈夫曼编码生成算法
void Huffmancode(Node* root, char code[], int depth)

核心逻辑（前序遍历）：
从根节点开始，向左子树走时记录0，向右子树走时记录1
遇到叶子节点时，将当前路径的0/1序列作为该字符的哈夫曼编码
递归遍历左右子树，生成所有字符的编码


四、业务流程设计

输入文本：支持字符串输入（可扩展为文件读取）
频率统计：得到每个字符的出现次数
构建哈夫曼树：通过优先队列完成节点合并
生成编码：递归遍历树，记录路径生成编码
输出结果：打印每个字符对应的哈夫曼编码


五、难点与解决方法
1. 优先队列的有序插入与出队

难点：保证每次取出的都是权值最小的节点
解决方法：
enqueue：采用插入排序的思想，将新节点按权值从后往前插入到合适位置，保持队列升序
dequeue：取出队首元素后，将后续元素整体前移一位

2. 哈夫曼树的构建逻辑

难点：中间节点的权值合并与队列维护
解决方法：合并两个节点后，生成新的中间节点，再调用enqueue按权值插入队列，保证队列始终有序

3. 递归生成哈夫曼编码

难点：如何记录当前路径的编码序列
解决方法：用code数组存储当前路径的编码，depth记录当前节点的深度，递归返回时自动回溯修改编码（code[depth]会被后续递归覆盖）

4. 代码中的关键疑问解答

为什么buildhuffmanTree中最后return dequeue(q)，
当列表没有元素的时候，会不会返回空？导致没有返回到这棵树的根节点

答案：其实不会，因为这每次循环取出两个节点，再返还一个节点，每轮循环只减少一个节点，最终剩下的那个节点个数一定是1，且一定是根节点。
